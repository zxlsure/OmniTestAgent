# ARCH-003: 资源泄漏

## 现象
应用退出时数据库、日志、子进程等资源未正确释放。

## 具体位置
1. `closeDatabase()` 存在但从未被调用，数据库定时保存器不清理，内存数据可能丢失
2. `logger.init()` 从未被调用，`logStream` 始终为 null，文件日志完全失效
3. McpService 启动的 ChildProcess 无退出清理机制
4. LlmService 缓存的 OpenAI client 无 destroy 方法

## 改进方案
1. 在 `app.on('before-quit')` 中调用 `closeDatabase()` 和 `logger.destroy()`
2. 应用启动时调用 `logger.init()`
3. McpService 增加 `disconnect()` 方法
4. 应用退出流程：McpService.disconnect() → closeDatabase() → logger.destroy()

## 经验教训
1. **每个获取资源的操作必须配对释放操作**：open/close、init/destroy、connect/disconnect
2. **Electron 应用的退出钩子**：`before-quit` 事件是释放资源的最后机会
3. **死代码是信号**：有 close/destroy 方法但未被调用，说明初始化-释放配对不完整
