# BUG-004: dev.js 端口检测失败导致 Electron 双启动

## 现象
运行 `node scripts/dev.js` 后，出现两个 Electron 实例，第二个日志输出 "Warning: Could not detect Vite URL, using default..."。

## 根因分析
三个问题叠加：
1. Vite 输出包含 ANSI 转义码（如 `\x1b[32m`），正则 `127\.0\.0\.1:\d+` 匹配时被 ANSI 码干扰
2. `setInterval` 的 URL 检测成功后 `clearInterval`，但 15 秒超时的 `setTimeout` 回调仍然执行
3. 超时回调未检查 Electron 是否已启动，直接再次调用 `startElectron()`

## 修复方案
1. 解析 URL 前剥离 ANSI 转义码：`viteOutput.replace(/\x1b\[[0-9;]*m/g, '')`
2. 添加 `electronStarted` 标志位，`startElectron()` 中检查防止重复启动
3. 超时回调中检查标志位，仅在未启动时回退

## 经验教训
1. **终端输出解析必须处理 ANSI 码**：彩色终端输出包含转义序列，直接正则匹配不可靠
2. **异步并发控制需防重入**：多个异步回调（interval + timeout）可能同时触发，需要标志位或锁
3. **更好的方案**：让 Vite 输出到 pipe，使用 `--json` 或自定义插件输出结构化端口信息

## 影响范围
- 文件：`scripts/dev.js`
