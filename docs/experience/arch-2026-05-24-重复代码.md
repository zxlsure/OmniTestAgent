# ARCH-004: 重复代码

## 现象
多处相同或高度相似的代码模式，修改一处容易遗漏另一处。

## 具体位置
1. **4个Skill execute() 模板**：requirementParser、caseGenerator、testDesigner、scriptGenerator 结构几乎完全相同，仅 prompt 和 outputKey 不同
2. **2个Bot客户端**：qqBotClient.send() 与 welinkBotClient.send() 仅 headers/body 不同，testConnection() 完全相同
3. **3个ConfigRepo upsert**：LlmConfigRepo、McpConfigRepo、ChannelConfigRepo 的 save() 方法模式相同
4. **2处动态导入**：secureStore.ts 和 LlmService.ts 的 `new Function('module', 'return import(module)')` 完全重复
5. **9处创建模式**：所有 Repository 的 `randomUUID() + new Date().toISOString()` 重复

## 改进方案
1. 抽取 BaseSkill 基类，子类仅提供 prompt/outputKey/artifactType
2. 抽取 BaseBotClient 基类，子类仅提供 headers/body 构造
3. 抽取 BaseConfigRepo 基类，泛型化 upsert 逻辑
4. 抽取 dynamicImport 工具函数到 utils/dynamicImport.ts
5. Repository 基类提供 generateId() 和 now() 方法

## 经验教训
1. **三次重复就应抽象**：DRY 原则不仅是减少代码量，更是减少修改遗漏的风险
2. **模板方法模式**：基类定义算法骨架，子类仅实现差异化部分
3. **工具函数集中管理**：通用模式提取到 utils，避免散落在各模块
