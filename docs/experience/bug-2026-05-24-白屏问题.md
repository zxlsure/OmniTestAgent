# BUG-001: 渲染进程白屏

## 现象
Electron 窗口启动后显示空白页面，DevTools 中无 Vue 应用挂载迹象。

## 根因分析
`index.html` 中 `<script type="module" src="/src/main.ts">` 使用了绝对路径 `/src/main.ts`，但 `vite.config.ts` 已将 `root` 设为 `src/renderer`，Vite 从 `src/renderer` 目录提供文件，因此 `/src/main.ts` 路径不存在。

## 修复方案
将 `src` 改为相对路径 `./main.ts`，让 Vite 从 root 目录解析。

```html
<!-- 修复前 -->
<script type="module" src="/src/main.ts"></script>
<!-- 修复后 -->
<script type="module" src="./main.ts"></script>
```

## 经验教训
1. **Vite root 配置与 HTML 路径耦合**：修改 `vite.config.ts` 的 `root` 后，必须同步检查 `index.html` 中的所有资源路径
2. **electron-vite 的默认行为**：electron-vite 默认将 root 设为 `src/renderer`，手动配置时容易遗漏路径同步
3. **调试白屏的步骤**：先检查 DevTools Console 是否有模块加载失败错误，再检查 Network 面板的 404 请求

## 影响范围
- 文件：`src/renderer/index.html`
- 所有使用 Vite dev server 的开发环境
