# BUG-002: Arco Design 图标不显示

## 现象
所有使用 `<icon-xxx />` 的 Arco Design 图标组件在页面上不显示，控制台可能有 "Failed to resolve component" 警告。

## 根因分析
`main.ts` 中只注册了 `app.use(ArcoVue)`（主组件库），未注册 `@arco-design/web-vue/es/icon`（图标包）。Arco Design Vue 的图标组件不会随主包自动全局注册。

## 修复方案
在 `main.ts` 中增加图标包的全局注册：
```typescript
import ArcoVueIcon from '@arco-design/web-vue/es/icon'
app.use(ArcoVueIcon)
```

## 经验教训
1. **Arco Design Vue 图标需单独注册**：这与 Element Plus（自动导入）等框架不同，Arco 需要显式注册
2. **影响7个文件**：StatusIcon、TopNavBar、ChatView、TestExecutionView、KnowledgeView、ProjectManagementView、SkillsView
3. **动态组件 `<component :is="item.icon" />` 需要全局注册**：局部注册的组件无法通过字符串名称动态渲染

## 影响范围
- 文件：`src/renderer/main.ts`（修复点）
- 影响所有使用图标的组件
